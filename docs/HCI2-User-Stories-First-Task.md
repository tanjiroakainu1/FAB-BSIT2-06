# HCI 2 — First Task: User Stories

**Project:** Food Ordering Hermanas (Food Ordering System)  
**Template:** As a [type of user], I want to [goal], so that [reason].

---

## Guest

| # | User story |
|---|------------|
| 1 | As a **guest**, I want to **browse the menu without logging in** so that **I can see what is available before creating an account**. |
| 2 | As a **guest**, I want to **search products by name or description** so that **I can find specific dishes quickly**. |
| 3 | As a **guest**, I want to **filter the menu by category** so that **I can focus on main dishes, drinks, or desserts**. |
| 4 | As a **guest**, I want to **see a clear link to register or login** so that **I can place an order when I am ready**. |
| 5 | As a **guest**, I want to **use the forgot password page to check my request status** so that **I can recover my account after submitting proof**. |

---

## Customer

| # | User story |
|---|------------|
| 1 | As a **customer**, I want to **view product details (image, price, half/full tray, add-ons)** so that **I know exactly what I am ordering**. |
| 2 | As a **customer**, I want to **add a product to cart with optional half tray and notes** so that **my order matches my preference**. |
| 3 | As a **customer**, I want to **add add-ons (with half or full tray when available)** so that **I can order extras or combos in one go**. |
| 4 | As a **customer**, I want to **see my cart with main items and add-ons in separate sections with images** so that **I can review my order clearly**. |
| 5 | As a **customer**, I want to **change quantity or remove items in the cart** so that **I can correct my order before checkout**. |
| 6 | As a **customer**, I want to **enter order notes and payment reference at checkout** so that **special requests and payment proof are recorded**. |
| 7 | As a **customer**, I want to **place my order and see a clear confirmation** so that **I am sure the system received my order**. |
| 8 | As a **customer**, I want to **view my order history with item photos and status** so that **I can track past orders**. |

---

## Admin

| # | User story |
|---|------------|
| 1 | As an **admin**, I want to **add, edit, and delete categories** so that **the menu is organized for customers**. |
| 2 | As an **admin**, I want to **add and edit products (single or combo, half tray option, images)** so that **the menu is up to date**. |
| 3 | As an **admin**, I want to **move deleted products to archive and restore or permanently delete them** so that **I can manage the product list safely**. |
| 4 | As an **admin**, I want to **view and update order status, payment, and delivery** so that **orders are processed correctly**. |
| 5 | As an **admin**, I want to **view forgot-password requests and proof of ownership** so that **I can approve or reject fairly**. |
| 6 | As an **admin**, I want to **chat with kitchen and delivery staff (group and direct)** so that **we coordinate without leaving the system**. |

---

## Kitchen Staff

| # | User story |
|---|------------|
| 1 | As **kitchen staff**, I want to **see pending orders in one place** so that **I know what to prepare**. |
| 2 | As **kitchen staff**, I want to **mark an order as ready** so that **delivery knows it can be picked up**. |
| 3 | As **kitchen staff**, I want to **chat with admin and delivery** so that **I can clarify or report issues**. |

---

## Delivery Staff

| # | User story |
|---|------------|
| 1 | As **delivery staff**, I want to **see active deliveries** so that **I know which orders to deliver or hand over**. |
| 2 | As **delivery staff**, I want to **mark an order as delivered or picked up** so that **the order is closed correctly**. |
| 3 | As **delivery staff**, I want to **view delivery history** so that **I can refer to past completed orders**. |
| 4 | As **delivery staff**, I want to **chat with admin and kitchen** so that **we can coordinate when needed**. |

---

*All user stories follow the format: As a [type of user], I want to [goal], so that [reason].*
