# HCI 2 — Gemini Chat Prompt (Copy-Paste This)

Copy the entire block below and paste it into Gemini chat when you need help with your HCI 2 Usability Testing and Evaluation project.

---

## Prompt (copy from here)

```
I am working on Human Computer Interaction 2: Usability Testing and Evaluation. My project is a collaborative one where I already have a proposed system. My focus is NOT on coding but on:

- User-centered design
- Prototyping
- Documentation
- Usability testing
- Evaluation and improvement

**My system: Food Ordering Hermanas (Food Ordering System)**

- **Brief description:** Web-based food ordering: customers browse menu, add items (half/full tray, add-ons), place orders; staff (admin, kitchen, delivery) manage products, orders, and chat.
- **Target users:** Guest (browse only), Customer (registered, places orders), Admin, Kitchen staff, Delivery staff.
- **Main features:** Browse menu, search products, category filters, product details with add-ons and half/full tray, cart (main items vs add-ons with images), checkout (order notes, payment, reference), forgot password monitoring; admin: categories, products, archive, orders, forgot-password requests, chat; kitchen: orders, mark ready, chat; delivery: active deliveries, mark done, history, chat.

**Project objective:** To evaluate and improve this existing system using usability testing methods and proper HCI documentation.

I need help with the following structure:

**Phase 1: Understanding the Proposed System**
- System overview (brief description, target users, purpose, main features)
- Problem identification (usability problems users might experience; user needs to prioritize)

**Phase 2: User-Centered Design Documentation**
- User personas (at least 2): Name (fictional), Age, Occupation, Goals, Frustrations, Technical skills
- User stories in this format: "As a [type of user], I want to [goal], so that [reason]."
- User journey map: steps, user actions, thoughts, emotions, pain points
- Task flow diagram: step-by-step process of completing a major task

**Phase 3: Prototype / Mockup**
- Wireframe or mockup description for at least 5–7 screens
- Must reflect HCI principles: consistency, visibility, feedback, simplicity, etc.

**Phase 4: Usability Testing**
- Testing plan: 3–5 participants, list of tasks, method (Observation, Think-Aloud, Survey)
- What to record: errors, time to complete tasks, confusion points, user comments
- Evaluation and recommendations: identify usability issues, suggest design improvements, explain how improvements follow HCI principles

**FIRST TASK:** Create user stories based on the template: "As a [type of user], I want to [goal], so that [reason]." for all user types in my system (Guest, Customer, Admin, Kitchen staff, Delivery staff).

Please help me generate or improve content for [specify which part: e.g. Phase 1, User Stories only, User Journey Map, Testing Plan, etc.].
```

---

## End of prompt

**Tip:** Replace the last line with the specific part you need, e.g.:
- "Please generate Phase 1 System Overview and Problem Identification."
- "Please generate user stories only for Customer and Admin."
- "Please create a user journey map for a customer placing an order."
